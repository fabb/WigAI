# Story 4.2: Implement Preferences-Backed Configuration

**Epic:** Epic 4 – Extension UI Settings Integration  
**Story ID:** 4.2  
**Priority:** High  
**Estimated Effort:** 1 day  
**Status:** Planned

## User Story

As a WigAI user, I want to edit the MCP host and port directly inside Bitwig’s Controller Preferences so I never have to modify configuration files manually.

## Detailed Requirements

1. Implement `PreferencesBackedConfigManager` that satisfies `ConfigManager` and persists values through `host.getPreferences()`.
2. Create a “Network Settings” section within the WigAI controller preferences that contains:
   - MCP Host (`SettableStringValue`, max 50 chars, default `localhost`)
   - MCP Port (`SettableRangedValue`, range 1024–65535, default `61169`)
   - Read-only Connection URL string showing `http://{host}:{port}/sse`
3. Display a short instructional message reminding users to share the URL with external AI agents.
4. Validate inputs and fall back to defaults when values are missing or out of range, logging a warning when this happens.
5. Do **not** restart the MCP server yet—this story only surfaces and persists settings.

## Technical Notes

- Use the Bitwig preferences API (`host.getPreferences()`) and store the latest host/port in member fields so `ConfigManager` getters return current values.
- The Connection URL field can be built via `String.format` and may be disabled/read-only.
- Reuse the legacy configuration as a fallback only during initialization; once preferences load successfully they should be the single source of truth.

## Acceptance Criteria

1. The WigAI preferences panel shows editable MCP Host/Port fields and a read-only Connection URL field.
2. Host/port values persist across Bitwig restarts and are returned by `ConfigManager`.
3. Invalid inputs (blank host, out-of-range port) revert to defaults with warning logs.
4. Instructional text is visible to the user within the preferences UI.
5. No MCP server restart or observer logic is triggered in this story.
