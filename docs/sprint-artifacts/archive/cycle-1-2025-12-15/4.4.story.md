# Story 4.4: WigAI Extension Integration & Help Surfacing

**Epic:** Epic 4 – Extension UI Settings Integration  
**Story ID:** 4.4  
**Priority:** Medium  
**Estimated Effort:** 1-2 days  
**Status:** Planned

## User Story

As a WigAI user, I want the extension to reflect the active connection URL everywhere (preferences, logs, help entry point) so I can quickly connect external AI agents with confidence.

## Detailed Requirements

1. Update `WigAIExtension.init()` to instantiate `PreferencesBackedConfigManager`, register the MCP server observer, and remove the legacy configuration implementation.
2. Ensure the read-only Connection URL in preferences updates instantly when host/port change.
3. Log the active connection URL and configuration source during extension startup.
4. Confirm `WigAIExtensionDefinition.getHelpFilePath()` returns the GitHub README, enabling Bitwig’s Help button to open current documentation.
5. Run through a regression checklist to confirm transport commands, MCP tools, and other extension features still function.

## Technical Notes

- After this story, `PreferencesBackedConfigManager` should be the sole implementation shipped with WigAI.
- Consider adding a small helper for formatting the connection URL so both logs and UI reuse the same logic.
- Capture any migration notes for users upgrading from older versions (e.g., reading existing config before migrating to preferences).

## Acceptance Criteria

1. WigAI logs on startup that it is using preferences-backed configuration and prints the connection URL.
2. The Connection URL field in preferences updates immediately after host/port edits (verified manually).
3. Clicking the Help button in Bitwig’s Controller Preferences opens `https://github.com/fabb/WigAI/blob/main/README.md`.
4. Manual smoke tests confirm MCP commands still function with the new configuration plumbing.

## Definition of Done

- [ ] Legacy configuration code paths removed from production builds.
- [ ] Manual verification recorded (screenshots or short notes) for preferences updates and Help button behavior.
- [ ] All automated tests pass.
- [ ] Code review completed.

## Dependencies

- Stories 4.1 through 4.3 must be completed so the new manager, preferences UI, and restart flow exist before integrating them.
